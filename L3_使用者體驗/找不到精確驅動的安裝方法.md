## 1. 連接埠選擇：9100、RAW、LPR、IPP 是什麼

安裝**網路印表機**時，系統會要求選擇「連接埠類型」。這決定電腦用什麼協定把資料送給印表機。

### 各協定比較

| 協定 | 預設 Port | 說明 | 適用場景 |
|------|----------|------|---------|
| **RAW（Port 9100）** | TCP 9100 | 直接傳送原始列印資料，不加包裝 | **最常用**，絕大多數網路印表機 |
| **LPR / LPD** | TCP 515 | 早期 Unix/Linux 標準協定 | 舊型印表機、部分 Linux 列印伺服器 |
| **IPP** | TCP 631 | 現代標準，支援狀態回報、加密 | 現代印表機、雲端列印 |
| **SMB（\\共用）** | TCP 445 | Windows 共用印表機 | 透過另一台電腦共享的印表機 |

### RAW / Port 9100 詳解（最常遇到）

```
電腦  →  TCP Port 9100  →  印表機
       （直接傳原始資料）
```

- **RAW** 不是一種特殊格式，而是「原始資料直送」的意思
- 印表機收到後直接解譯（PCL、ESC/P、ZPL 等）
- 幾乎所有品牌的網路印表機都支援
- **安裝時選 Standard TCP/IP Port，Protocol 選 RAW，Port 填 9100** 即可

### LPR 詳解

```
電腦  →  TCP Port 515  →  印表機（需填 Queue Name）
```

- 需要額外填寫「佇列名稱（Queue Name）」，通常填 `lp` 或 `auto` 或空白
- 比 RAW 多一層握手確認，速度略慢
- 若 Port 9100 無效，可嘗試 LPR

### 何時選哪個？

```
優先嘗試順序：RAW(9100) → LPR(515) → IPP(631) → SMB 共用
```

> 大多數狀況下，**RAW Port 9100** 都能成功。若列印沒反應，再嘗試 LPR。

## 2. USB 連線 vs 網路連線的差異

### 差異對照表

| 比較項目 | USB 連線 | 網路連線（IP） |
|---------|---------|--------------|
| 安裝難度 | 低（隨插即用） | 中（需知道 IP） |
| 線材限制 | 距離 ≤ 5 公尺 | 不限距離（在同網段即可） |
| 多人共用 | 只有插著的那台電腦能印 | 同網路所有人都可連 |
| 斷線情形 | 拔線即斷 | 印表機關機或 IP 變動才斷 |
| 驅動安裝 | 插入後 Windows 通常自動偵測 | 需手動建立 TCP/IP 連接埠 |
| 速度 | USB 2.0 ≈ 480 Mbps（足夠） | 100 Mbps 網路（足夠） |
| 適合場景 | 個人專用、臨時使用 | 多人共用、固定工作站 |

### USB 安裝特點

- Windows 插入後會嘗試自動搜尋驅動（Windows Update）
- 找不到時，裝置管理員會顯示「未知裝置」或「!」
- 此時手動安裝驅動：裝置管理員 → 右鍵 → 更新驅動程式 → 瀏覽我的電腦

### 網路安裝特點

- 必須先確認**印表機 IP**（參考：[找不到印表機 IP 怎麼辦](Printer_Installation_Guide.md#2-找不到印表機-ip-怎麼辦)）
- 安裝路徑：**控制台 → 裝置和印表機 → 新增印表機 → 新增本機印表機 → 建立新連接埠 → Standard TCP/IP Port**

> **重要**：選「新增本機印表機」（不是「新增網路印表機」）才能手動建立 TCP/IP Port。選「新增網路印表機」是走 Windows 自動搜尋，在非 Domain 環境常常找不到。

## 3. 找不到驅動時的替代方案

### 方案一：使用相容型號驅動（最常用）

同廠牌同系列的舊型號驅動通常相容。

| 找不到驅動的型號 | 可嘗試的替代驅動 |
|----------------|---------------|
| EPSON LQ-3500II | EPSON LQ-3500 / LQ-2500 |
| EPSON WF-T810 | EPSON WF-7820 / WF-7840 |
| TSC TTP-245C | TSC TTP-244 / TTP-343 |
| SATO CG408 | SATO CG412 / CG212 |
| 任何 HP 噴墨 | HP Deskjet 系列通用驅動 |

**尋找相容型號的方法**：
1. Google「`[型號] compatible driver`」或「`[型號] 替代驅動`」
2. 查廠商官網的「相容機型」列表
3. 看印表機底部標籤上的「PCL 版本」或「ESC/P」字樣，找支援相同語言的驅動

---

### 方案二：Windows 內建通用驅動

不需下載，直接從 Windows 驅動清單選取。

**適用情境**：點陣印表機、簡單文字列印、急用時。

**步驟**：
1. 新增印表機 → 選擇「從清單選取」
2. 廠商選 **Generic** 或對應品牌
3. 型號選：
   - `Generic / Text Only`：純文字，無格式，最保險
   - `Generic IBM Graphics 9pin`：9 針點陣機
   - `Generic IBM Graphics 24pin`：24 針點陣機（如 LQ 系列）
   - `HP LaserJet`：支援 PCL 的雷射印表機

> 限制：通用驅動無法使用印表機特定功能（如紙匣選擇、雙面列印），但基本列印通常可行。

---

### 方案三：製造商通用驅動包

各大廠牌提供跨型號通用驅動：

| 廠牌 | 通用驅動名稱 | 說明 |
|------|------------|------|
| EPSON | Epson Universal Print Driver | 支援大多數 EPSON 雷射/噴墨 |
| HP | HP Universal Print Driver (UPD) | 支援幾乎所有 HP 雷射印表機 |
| Canon | Canon Generic Plus PCL6 | 支援 Canon 辦公印表機 |
| TSC | TSC Universal Driver | 支援所有 TSC 標籤機 |

---

### 方案四：Manufacturer Class Driver（Windows 內建廠牌驅動）

Windows 10/11 有內建部分廠牌的基本驅動，透過 Windows Update 取得：

1. 新增印表機時選「Windows Update」讓系統自動搜尋
2. 或：**設定 → Windows Update → 進階選項 → 選用更新 → 驅動程式更新**

---

### 方案五：使用 IPP（無驅動列印）

Windows 10/11 支援透過 IPP 協定「無驅動列印」，印表機需支援 AirPrint 或 IPP Everywhere。

1. **設定 → 藍牙與裝置 → 印表機與掃描器 → 新增裝置**
2. 點選「我要的印表機不在清單中」
3. 選「使用 IP 位址或主機名稱新增印表機」
4. 裝置類型選「IPP 裝置」
5. 輸入 `ipp://<印表機IP>/ipp/print`
6. Windows 自動下載相容驅動

## 4. 用通用驅動安裝的完整步驟

### 網路印表機（TCP/IP + 通用驅動）

```
控制台
 └─ 裝置和印表機
     └─ 新增印表機
         └─ 我要的印表機不在清單中
             └─ 使用 TCP/IP 位址或主機名稱新增印表機
                 ├─ 裝置類型：TCP/IP 裝置
                 ├─ 主機名稱或 IP 位址：[輸入印表機 IP]
                 ├─ 連接埠名稱：自動填入（可保留）
                 └─ [下一步]
                     └─ 通訊協定：RAW　連接埠號碼：9100
                         └─ [下一步]
                             └─ 選取驅動程式
                                 ├─ 廠商：[選品牌] 或 Generic
                                 └─ 型號：[選相容型號]
```

### 驗證連接埠是否通

安裝前先確認 Port 9100 可達：

```cmd
:: 確認印表機 IP 可 ping 到
ping 192.168.1.51

:: 確認 Port 9100 開啟（需 Windows 10/11）
Test-NetConnection -ComputerName 192.168.1.51 -Port 9100
```

若 `TcpTestSucceeded : True` 表示可連，繼續安裝。
若 `False`，嘗試改用 Port 515（LPR）或確認防火牆設定。

## 5. 驗證安裝是否成功

1. **列印測試頁**：印表機 → 右鍵 → 印表機屬性 → 列印測試頁
2. **確認狀態顯示「就緒」**：不是「離線」或「錯誤」
3. **從記事本列印**：開啟記事本，輸入幾個字，Ctrl+P → 選該印表機 → 列印
4. **若測試頁亂碼**：驅動語言不符，嘗試換另一個相容驅動型號

> 亂碼通常代表驅動語言（PCL/ESC-P/PostScript）與印表機不符，並非 IP 或 Port 問題。
